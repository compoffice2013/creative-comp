## バイブコーディングとは
ノリと対話でコーディングして、ホームページやアプリケーションを作成する開発スタイルです。

* 人工知能エージェントと会話しながら進める
* 「まず動かす → 直す → 仕上げる」の反復で完成へ近づける

## グーグルアンチグラビティとは
人工知能エージェントを搭載した開発プラットフォームです。

* マイクロソフト社の Visual Studio Code（ビジュアル スタジオ コード）をベースにした派生エディター
* Cursor（カーソル）や Windsurf（ウインドサーフ）と同系統の人工知能搭載エディター
* 人工知能エージェントとの対話を中心に開発を進める
* 活用方法は無限大

## グーグルアンチグラビティの基本操作
基本は「依頼」と「質問」でポチポチ。

* 人工知能エージェントに依頼する
* 人工知能エージェントに質問する

## 事前準備

### Git（ギット）のインストール
**目的**
アプリケーションのデータや変更履歴を管理するための基本ツールです。
研修中は操作を意識しなくても大丈夫です。インストールされていればOKです。

**対象リンク**
https://git-scm.com/

**手順**
1. 上記リンクを開く
2. Download（ダウンロード）をクリック
3. Windows は Windows 版、macOS は macOS 版を選択
4. ダウンロード後は基本的に「次へ」を押して完了

### Node.js（ノード ジェイエス）のインストール
**目的**
アプリケーションを動作させるための基盤となるソフトウェアです。使い方を覚える必要はありません。

**対象リンク**
https://nodejs.org/

**手順**
1. 上記リンクを開く
2. 表示されているダウンロードボタンをクリック
3. 画面の案内に従ってインストール（基本は「次へ」でOK）

### Next.js（ネクスト ジェイエス）のインストール
**目的**
WebサイトやWebアプリケーションの画面（フロントエンド）を効率よく作成するためのフレームワーク（土台）です。これから作成するアプリのメインとなる部分です。

**対象リンク**
https://nextjs.org/

**手順**
※Next.jsはWebサイトからのダウンロードではなく、ターミナル（コマンドプロンプト）からコマンドを入力してインストールします。事前にNode.jsがインストールされている必要があります。

1. ターミナル（Mac）または コマンドプロンプト / PowerShell（Windows）を開く
2. アプリケーションを作成したいフォルダに移動する
3. 以下のコマンドをコピー＆ペーストしてEnterキーを押す（ここでは例としてバージョン15を指定しています）
   `npx create-next-app@15 my-app`
   ※「my-app」の部分は好きなアプリの名前に変更できます。
   ※常に最新バージョンを使いたい場合は、15 の部分を latest にして `npx create-next-app@latest my-app` と入力します。
4. 画面にいくつか英語で質問（TypeScriptを使うか？など）が表示されるので、案内に従って選択する（基本はそのままEnterキーを押して「Yes / デフォルト設定」で進めてOK）

### GitHub（ギットハブ）にログイン
**目的**
作成したアプリケーションのプログラムコードを保存するクラウド上の保管場所です。

**対象リンク**
https://github.com/

**手順**
1. 上記リンクを開く
2. アカウントがない場合は Sign up（サインアップ）で新規登録
3. すでにある場合は Sign in（サインイン）でログイン

### Vercel（バーセル）にログイン
**目的**
作成したアプリケーションをインターネット上に公開するためのサービスです。

**対象リンク**
https://vercel.com/

**手順**
1. 上記リンクを開く
2. Sign up（サインアップ）または Log in（ログイン）
3. GitHub（ギットハブ）アカウントでログイン

### Vercel（バーセル）と GitHub（ギットハブ）を連携させる
**目的**
GitHub にプッシュした内容を自動的に公開できるようにする設定です。
一度設定すれば、その後は意識する必要はありません。

**手順**
1. Vercel にログイン
2. Add New（アド ニュー）→ Project（プロジェクト）をクリック
3. GitHub を選択
4. GitHub の認証画面が表示されたら連携を許可
※途中で「すべてのリポジトリを許可するか」と聞かれた場合は、すべて許可を選択してください。

### Gemini API（ジェミニ エーピーアイ）の取得
**目的**
Gemini（ジェミニ）を利用するための鍵となる API Key（エーピーアイ キー）です。
研修中は仕組みを深く理解する必要はありません。取得して控えておけばOKです。

**対象リンク**
https://aistudio.google.com/

**取得手順**
1. 上記リンクを開く
2. Google（グーグル）アカウントでログイン
3. Get API key（ゲット エーピーアイ キー）または API キーを作成 をクリック
4. 新しい API キーを作成
5. 表示された API キーをコピーして控える

**注意点**
* API Key（エーピーアイ キー）はパスワードと同じ扱い
* GitHub や資料に直接記載しない
* 他人に共有しない

### 画面構成の説明
大きく2つのビューがあります。

* **エージェントマネージャービュー（エージェントモード）**

## 実際にアプリケーションを作成する

### 今回作るもの
**時計アプリケーション**
アナログ表示とデジタル表示を切り替えられる、おしゃれな時計です。

### 手順
Start new conversation → Playground

**推奨設定**
* コーディングは Fast（ファスト）ではなく Planning（プランニング）
* 言語モデルは Gemini 3 Pro（ジェミニ スリー プロ）を使用

**入力するプロンプト（参考）**
> 時計アプリケーションを作成してください。
> アナログ表示とデジタル表示を切り替えられる、おしゃれな時計です。
> ユーザーインターフェースはすべて日本語でお願いします。
> 時間に応じたAIのメッセージを表示してください。

### 公開までの作業手順

**1. GitHub（ギットハブ）へプッシュ**
1. GitHub → New（ニュー）
2. Repository name（リポジトリ ネーム）を入力
3. Private（プライベート）を選択
4. Create repository（クリエイト リポジトリ）
5. リポジトリのアドレスをコピー
6. グーグルアンチグラビティに「このアドレスにプッシュしてください」と依頼

**2. Vercel（バーセル）で環境変数を設定**
1. Settings（セッティングズ）
2. Environment Variables（エンバイロメント バリアブルズ）
3. Add Environment Variables（アド エンバイロメント バリアブルズ）
4. Key（キー）に名前を入力
5. Value（バリュー）に Gemini API の API Key を登録

**3. Vercel（バーセル）でデプロイメント**
1. GitHub からインポート
2. デプロイメント（公開）

## 人工知能エージェントとの会話のヒント
* 作業が止まったと感じたら、新しい会話で「作業は進んでいますか？」と質問する
* エラーが出たら、新しい会話で「先ほどエラーが出ました」と伝える
* うまくいかない場合は使用する大規模言語モデルを変更してみる
* 仕上がり確認は「ブラウザで確認したい」と依頼する
* ネイティブアプリケーションのように使いたい場合は、「プログレッシブウェブアプリケーション化してください。ファビコンとアイコンを設定してください」と伝える

## 人工知能エージェント ＆ GitHub 用語集（カタカナ読み付き）

### 人工知能エージェント操作でよく出る用語
* **Accept（アクセプト）**
  意味：内容を了承して処理を進める。迷った場合はこれ。
* **Proceed（プロシード）**
  意味：現在の内容で処理を継続する。
* **Dismiss（ディスミス）**
  意味：閉じる。今回は実行しない。
* **Try it out（トライ イット アウト）**
  意味：試しに実行してみる。安全なお試し操作。
* **Reject（リジェクト）**
  意味：拒否する。提案や操作を受け入れず実行しない。
* **Authorize（オーソライズ）**
  意味：権限を許可する。GitHub 連携時によく表示される。
* **Approve（アプルーブ）**
  意味：承認する。連携や操作を正式に認める。
* **Apply（アプライ）**
  意味：設定や変更内容を反映する。
* **Discard（ディスカード）**
  意味：変更を保存せずに破棄する。
* **Deploy（デプロイ）**
  意味：アプリケーションを公開状態にする。

### API・設定関連用語
* **API（エーピーアイ）**
  意味：サービス同士を接続するための仕組みや取り決め。
* **API Key（エーピーアイ キー）**
  意味：API を利用するための本人確認用の鍵。
* **Environment Variable（エンバイロメント バリアブル）**
  意味：プログラムの外側で管理する設定値。秘密情報の管理に使う。
* **.env ファイル（ドット エンブ ファイル）**
  意味：環境変数を記述するための専用ファイル。GitHub には基本アップロードしない。
* **Secret（シークレット）**
  意味：GitHub やクラウドサービス側で管理する安全な秘密情報。

### GitHub・Git 関連用語
* **Repository（リポジトリ）**
  意味：アプリケーションやプログラムコードを保存する場所。
* **Push（プッシュ）**
  意味：変更内容を GitHub に送信する操作。
* **Pull（プル）**
  意味：GitHub の最新内容を自分の環境に取り込む操作。
* **Clone（クローン）**
  意味：リポジトリを自分の環境に丸ごとコピーする操作。
* **Commit（コミット）**
  意味：変更内容をひとまとまりとして記録する操作。
* **Staging（ステージング）**
  意味：コミット前に保存対象を選ぶ段階。
* **Branch（ブランチ）**
  意味：作業用の分岐。本体を壊さずに作業するための仕組み。
* **Pull Request（プル リクエスト）**
  意味：変更内容を提案し、確認してもらう仕組み。今回は基本的に使用しません。

## 研修用 超要約フレーズ
* 迷ったら Accept（アクセプト）か Proceed（プロシード）
* 連携時は Authorize（オーソライズ）
* API Key（エーピーアイ キー）は Environment Variable（エンバイロメント バリアブル）で管理
* やめたいときは Dismiss（ディスミス）
* GitHub の用語は意味と読みが分かれば十分
