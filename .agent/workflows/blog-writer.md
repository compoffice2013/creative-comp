---
description: イベントレポート（マジックショー・講演・ワークショップ等）からブログ記事とSNS投稿を一括生成する
---

このワークフローは、マジックショーや講演会、ワークショップなどのイベント終了後に、魅力的なブログ記事と各SNSプラットフォームに最適化された投稿を生成します。

## 手順

0. **クローンモード準備（必須）**
   - `_index.md` でワークスペース全体構造を確認
   - `00_persona/persona.md` でルール・禁止語・文体を確認
   - `04_knowledge/personal_essays/essay_themes.md` でテーマ別インデックスを確認
   - テーマに合ったエッセイを2〜3本読む
   - `04_knowledge/rinri/` から関連する倫理ブログがあれば参照
   - `04_knowledge/episodes.md` から再利用できるエピソードがあるか確認
   - `04_knowledge/quotes.md` から使えるフレーズを確認
   - これらを踏まえて「マジシャン・コンプレッサー」として書く

   - `04_knowledge/rank_math_seo_guidelines.md` で100点獲得の要件を確認

1. **文字起こしの分析**
   - 渡されたテキストから以下の要素を抽出します。
     - メインテーマ
     - 重要なトピック（3〜5点）
     - 話し手の感情や名言
     - リスナーへのメッセージ

2. **ブログ記事の生成（WordPress用 / Rank Math 100点対策）**
   - **構成**:
     - **SEO設定セクション**:
       - [フォーカスキーワード（カンマ区切り）]: 3〜4語含める
       - [SEOタイトル]: **数字**と**パワーワード**（驚愕、秘密、最高の、等）を含め、**キーワードを先頭**に配置。[文字数: 30-35文字程度]
       - [メタディスクリプション]: キーワードを含め、クリックしたくなる内容に。[120文字前後]
       - [パーマリンク(英語)]: キーワードを含み、**75文字以内**の短いスラッグ。
     - **タイトル**: 読者を惹きつけ、かつSEO要件（数字・パワーワード）を満たすもの。
     - **本文**:
       - **タイトル（H1）**: 記事の一番上に、選定したタイトルをH1（#）で記述する。
       - **導入文**: **最初の段落（冒頭10%）**に必ずフォーカスキーワードを自然に含める。
       - **見出し構成**: H2見出しの少なくとも1つにフォーカスキーワードを含める。
       - **段落**: 1段落は3〜4行以内（スマホでの可読性確保とSEO対策）。
       - **まとめ**: アクションを促す結び。
   - **トーン・文体（絶対遵守）**: 
     - **一人称（私、僕、自分等）を一切使用しない。**
     - 親しみやすく、かつ実況感のある語り口（「です・ます」調）
     - 強調（太字）を適度に使用。
     - **画像の指示**: 記事内に「<!-- 画像配置: ここに[キーワード]を含むALT属性の画像を4枚以上配置してください -->」というコメントを適所に挿入する（100点要件）。
   - **SEO**: 
     - キーワード出現率: 1.0%〜1.5%を目指して自然に盛り込む。

2.5. **SEO強化（内部リンク・外部リンク挿入）**
   - **ツール実行**: ターミナルで `node d:\Obsidian\creative-comp\scripts\wp_search.js [キーワード]` を実行し、関連する過去記事を検索する。
   - **リンク挿入**: 内部リンク（2〜3件）と外部リンク（会場サイト等）を本文中に組み込む。
   - **重要ルール**: リンクは必ず **完全なURL（https://...）** で記述すること。Obsidianの内部リンク形式（`[[...]]`）はWordPressで機能しないため **絶対に使用しない**。
     - エッセイへのリンク: 該当ファイルの YAML Frontmatter にある `url` フィールドの値を使用する。
   - **フォーマット**: リンクは**改行して単独の段落**として表示する（インラインではなく独立した行に配置）。
   - **SEOファイル**: フォーカスキーワード、タイトル、メタディスクリプション、パーマリンクのみを記載。リンクは記載しない。
   - **目的**: サイト内回遊率を高め、Rank Mathのスコアを100点に近づける。

3. **SNS投稿の生成**
   - **Facebook用**: 
     - ストーリー性のある紹介文（200〜400文字程度）
     - 「続きを読む」を誘うフックとリンク誘導
     - **空行処理**: Facebookは空行を詰めてしまうため、句点（。）ごとに改行し、段落間（。の直後）には必ず「⠀」（点字空白文字 U+2800）を1つだけ置いた空行を挿入してください。読点（、）では改行しないでください。
   - **X（旧Twitter）用**:
     - 140文字以内でインパクトを重視した要約
     - ハッシュタグ（2-3個）
     - **空行処理**: 段落間には「⠀」（点字空白文字 U+2800）を使用
   - **Instagram用**:
     - **ハッシュタグは使用しない**（最新のガイドラインに基づき、検索性を高めるキーワードをキャプション本文に自然に組み込む）
     - 視覚的な表現を意識したエモーショナルな導入。
     - 箇条書きを活用した重要ポイントの整理。
     - **空行処理**: 段落間には「⠀」（点字空白文字 U+2800）を使用

4. **成果物の保存と提示**
   - 生成した各コンテンツを以下のディレクトリにファイルとして保存してください（ファイル名は `YYYY-MM-DD-タイトル.md` 形式）。
     - ブログ記事: `02_blog/drafts/`（本文のみ。SEO設定は別ファイル）
     - Facebook投稿: `02_blog/facebook/`
     - X投稿: `02_blog/x/`
     - Instagram投稿: `02_blog/instagram/`
     - SEO要素: `02_blog/seo/`
   - **WordPressへの貼り付け方法**: `Ctrl+Shift+V`（書式なしペースト）を使用すること。通常の`Ctrl+V`ではMarkdownが正しく認識されない。
   - 保存後、各ファイルへのリンクを順番に表示してください。
     - [ブログ記事 (WordPress)](../02_blog/drafts/...)
     - [Facebook投稿](../02_blog/facebook/...)
     - [X投稿](../02_blog/x/...)
     - [Instagram投稿](../02_blog/instagram/...)
     - [SEO要素](../02_blog/seo/...)

5. **ブログ記事編集後のSNS・SEO更新（任意）**
   - ユーザーがブログ記事を編集した後、「SNSも更新して」または「反映して」と依頼された場合：
     1. 編集後のブログ記事（`02_blog/drafts/`）を読み込む
     2. 編集内容を踏まえて、以下を再生成・上書き保存する：
        - Facebook投稿（`02_blog/facebook/`）
        - X投稿（`02_blog/x/`）
        - Instagram投稿（`02_blog/instagram/`）
        - SEO要素（`02_blog/seo/`）- タイトルや内容に変更があれば更新
     3. 各プラットフォームのルール（手順3参照）を遵守すること
   - **注意**: 自動更新ではなく、ユーザーからの明示的な依頼があった場合のみ実行する


## ユーザーの固定設定（あなたの好み）
- **呼称とアイデンティティ**:
  - **本名は絶対に公開しない。** ブログやSNS等、すべての公の発信において「マジシャン・コンプレッサー」または「コンプさん」として振る舞う。
  - プロマジシャンであり、富山県倫理法人会 幹事長であるというアイデンティティを大切にする。
- **コア・フィロソフィー**:
  - **「芸は人なり」**: 芸の良し悪しは演じる人間の格に依存するという信念を、活動の根幹として読者に伝える。
- **プラットフォーム別ルール**:
  - **Facebook**:
    - **ハッシュタグは基本的に使用しない。**
    - 広告感を消すため「【見出し】」は使わず、共感を得やすい印象的な一文から始める。
    - 役職（幹事長など）に触れる際は、自慢や嫌味にならぬよう「たまたま拝命している」「先人の努力の賜物」といった謙虚な表現を徹底する。
    - **ブログへの誘導**: 投稿文末尾では直接リンクを貼らず、「ブログはコメント欄から」と簡潔に案内する。「是非読んでください」といったお願いの文章は使用しない。
    - **改行ルール**: 「、」では改行せず、「。」で改行し、さらに一行あける（「⠀」を入れた空行を作る）。長文でもこのルールを徹底し、視認性を高める。
    - 長さは300〜400文字程度の、エモーショナルで読みやすい音量を維持する。
  - **Instagram**:
    - **ハッシュタグは一切記述しない。** 本文中に重要なキーワードを文章として組み込む。
  - **WordPress**:
    - 箇条書きや強調、適度な改行を使い、スマホ読者が見やすい構成にする。
    - **記事の冒頭に必ずH1（#）でタイトルを記述する。**
    - 本文中の見出しはすべてH2（##）で統一する。
    - 強調のために「**」などのマークは使用しない（プレーンな読みやすさを重視）。
    - **過去記事・関連リンクの書式**: 
      - 必ず `[リンクテキスト](https://完全なURL)` の形式を使用する。
      - Obsidian内部リンク `[[...]]` は使用禁止。
      - ブロッククォート（`>`）は使わず、以下の形式で記載する：
      ```
      **👉 過去の研究会レポートはこちら**
      
      - [記事タイトル1](https://...)
      - [記事タイトル2](https://...)
      ```
- **トーン＆マナー**:
  - ポジティブ、情熱的、かつ謙虚。
  - 常に周囲や先人への「感謝」と「ご縁」を言葉にする。
  - 「富山を良くしたい」というリーダーとしての熱量を込める。
  - **表現の好み（ガイドライン）**:
    - 基本的に簡潔で力強い表現を好むが、文脈によっては強調表現も許容する。「極限まで〜」のような表現は、本当に必要な時以外は避ける傾向がある。
    - エピソードの引用において、記憶の曖昧さを表現することで「人間味」が出せる場合は、「うろ覚えですが」などのクッション言葉を活用する。これは常に必須ではなく、話の流れや雰囲気に合わせて臨機応変に判断すること。

## 注意事項
- 常に日本語で出力してください。
- 読者がその場にいたかのような臨場感を大切にしてください。
- 本名（高畑義光）が入力データに含まれていても、出力には絶対に出さないでください。
- このワークフローはユーザーの好みに合わせて継続的に改善されます。
- **ユーザー呼称**: コンプさんと呼ぶこと。完了報告は「コンプさん、◎◎の作業が完了しました！」の形式で。

## 完了時の作業（必須）
ブログ作業が完了したら、以下を必ず実行する：

1. **Obsidianでノート間リンクを作成**
   - 生成したブログ記事に関連するナレッジベースのノートがあれば、`[[ノート名]]` 形式でリンクを追加する
   - 例：保育園の記事なら `04_knowledge/` 内の関連エッセイへのリンクを検討

1.5. **ナレッジベースの資産化（任意提案）**
   - ブログ記事の中に「キラリと光るエピソード」があれば、`04_knowledge/episodes.md` への追記を提案する
   - 印象的なフレーズやお客様の反応があれば、`04_knowledge/quotes.md` への追記を提案する
   - FAQに使えそうな情報があれば、`04_knowledge/faq.md` への追記を提案する

2. **GitHubへコミット＆プッシュ**
   - 変更をステージング: `git add .`
   - コミット: `git commit -m "ブログ: [タイトルの要約]"`
   - プッシュ: `git push`

3. **完了報告**
   - 「コンプさん、◎◎の作業が完了しました！」と報告する
